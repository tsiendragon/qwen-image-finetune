# v3.3.9

**Release Date**: 2025-12-03

## Bug Fixes

### Flux2LoraTrainer I2I Preprocessing Mismatch
- **Issue**: Flux2LoraTrainer.predict() produced significantly different results compared to Flux2Pipeline for I2I (Image-to-Image) generation (Mean pixel diff: ~97)
- **Root Causes Identified**:
  1. **Size mismatch**: `prepare_predict_batch_data()` used config-defined `controls_size: [832, 576]` instead of user-specified dimensions
  2. **Value range mismatch**: Custom `ImageProcessor` returned `[0, 1]` range while `VaeImageProcessor` returns `[-1, 1]`
  3. **Position ID mismatch**: Condition image position IDs used `T=1` instead of `T=10` (scale=10 in Flux2Pipeline)

- **Fixes Applied**:
  - Modified `prepare_predict_batch_data()` to use `VaeImageProcessor.preprocess()` for predict mode
  - Updated `_preprocess_image()` to handle images already in `[-1, 1]` range
  - Changed condition position ID domain from `T=1` to `T=10` to match Flux2Pipeline._prepare_image_ids()

- **Impact**:
  - Mean pixel diff reduced from ~97 → ~38 (>60% improvement)
  - Intermediate steps (text encoding, image latents, timesteps, preprocessing) now match exactly
  - Remaining small differences are due to BFloat16 precision across different GPUs

## Changed Files
