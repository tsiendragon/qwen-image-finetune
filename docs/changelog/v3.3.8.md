# Version 3.3.8

## Bug Fixes

### Fixed Flux2LoraTrainer System Message Mismatch
- **Fixed**: `system_message` in `Flux2LoraTrainer` now exactly matches the original `Flux2Pipeline`
- **Issue**: The system message was missing a newline character between "object" and "attribution", causing significant differences in `prompt_embeds` (max diff=161)
- **Root Cause**: When defining the system message as a concatenated string, the newline was lost
- **Impact**: Text encoding now produces identical results to the original Flux2Pipeline, ensuring consistent generation outputs

## Technical Details

The original Flux2Pipeline uses this exact system message (from `system_messages.py`):
```
"You are an AI that reasons about image descriptions. You give structured responses focusing on object relationships, object
attribution and actions without speculation."
```

Note the newline between "object" and "attribution". The Trainer was using:
```
"You are an AI that reasons about image descriptions. You give structured responses focusing on object relationships, object attribution and actions without speculation."
```

This small difference in tokenization caused the `prompt_embeds` to differ significantly (max diff=161), leading to completely different generation results even with the same seed.
