# Version 3.3.5

## Enhancements

### Optimized Cross-Device Comparison in Comparison Script
- **Improved**: Comparison operations now automatically move tensors to the best available device (preferring GPU)
- **Feature**: Added `get_comparison_device()` helper function to intelligently select comparison device
- **Change**:
  - Both pipeline and trainer tensors are moved to the same device before comparison
  - Priority: GPU > CPU. If both are GPU, uses pipeline device
  - All comparison operations (latents, IDs) are performed on the same device
  - Comparison device is explicitly displayed in output
- **Impact**: Ensures accurate comparisons when using different devices, with optimal performance on GPU

## Technical Details

The comparison logic now:
1. **Device Selection**: Automatically selects the best device for comparison (prefers GPU)
2. **Tensor Movement**: Moves both pipeline and trainer tensors to the comparison device
3. **Type Consistency**: Ensures same dtype before comparison operations
4. **Clear Logging**: Displays which device is used for comparison

This ensures that when comparing results from different devices (e.g., `cuda:0` vs `cuda:1`), all comparison operations happen on the same device, preventing device mismatch errors and ensuring accurate results.
