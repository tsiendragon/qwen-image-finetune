# Version 3.3.6

## Enhancements

### Memory-Optimized Pipeline Loading in Comparison Script
- **Improved**: Optimized `load_pipeline()` function to reduce CUDA OOM errors
- **Feature**:
  - Added `check_gpu_memory()` helper function to monitor GPU memory usage
  - Pipeline components are now loaded to CPU first, then moved to GPU one by one
  - Memory is cleared after each component move to reduce peak memory usage
  - Added memory warnings when free memory is insufficient
  - Better error handling with helpful suggestions when OOM occurs
- **Change**:
  - Use `low_cpu_mem_usage=True` when loading pipeline
  - Move VAE, text_encoder, and transformer components sequentially
  - Clear CUDA cache after each component move
  - Display memory status at each step
- **Impact**: Significantly reduces memory peak during pipeline loading, making it more reliable when GPU memory is limited

## Technical Details

The optimized loading process:
1. **Memory Check**: Checks available GPU memory before loading
2. **CPU First**: Loads pipeline to CPU first (lower memory pressure)
3. **Sequential Move**: Moves components one by one (VAE → text_encoder → transformer)
4. **Cache Clearing**: Clears CUDA cache after each component to free fragmented memory
5. **Progress Tracking**: Shows memory usage at each step for debugging

This approach reduces the peak memory requirement from loading all components simultaneously to loading them sequentially, which is especially helpful when GPU memory is fragmented or partially occupied.
