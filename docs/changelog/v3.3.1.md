# Version 3.3.1

## Bug Fixes

### Flux2LoraTrainer Tokenizer Fix
- **Fixed**: `PixtralProcessor` tokenizer call in `Flux2LoraTrainer.encode_prompt()` method
- **Issue**: Tokenizer was called with `text` parameter but expected `messages` parameter for conversation format input
- **Change**: Updated tokenizer call from `text=formatted` to `messages=formatted` to correctly handle conversation-formatted prompts
- **Impact**: Fixes `ValueError: text input must be of type 'str'...` error when using T2I prediction mode

## Technical Details

The `_format_text_input()` method returns conversation-formatted messages (`list[list[dict]]`), which `PixtralProcessor` expects via the `messages` parameter, not `text`. This fix ensures proper tokenization of prompts for FLUX.2 text-to-image generation.
