# Version 3.3.4

## Enhancements

### Separate Device Support for Pipeline and Trainer in Comparison Script
- **Added**: Support for using different CUDA devices for Pipeline and Trainer in `compare_flux2_i2i.py`
- **Feature**: Users can now specify separate devices to avoid OOM errors when comparing models
- **Change**:
  - Added `--pipeline-device` argument to specify device for Flux2Pipeline
  - Added `--trainer-device` argument to specify device for Flux2LoraTrainer
  - Modified `load_trainer()` to accept device parameter and override config settings
  - Updated comparison functions to handle cross-device tensor operations
- **Impact**: Enables running comparison tests when one GPU is occupied, by distributing Pipeline and Trainer across different GPUs

## Technical Details

The script now supports:
1. **Separate device arguments**: `--pipeline-device cuda:0 --trainer-device cuda:1`
2. **Automatic device detection**: Falls back to `CUDA_VISIBLE_DEVICES` or defaults if not specified
3. **Device validation**: Checks availability and displays memory information for both devices
4. **Cross-device comparison**: Automatically handles tensor movement when comparing results from different devices

Usage examples:
```bash
# Use different GPUs for pipeline and trainer
python tests/src/trainer/compare_flux2_i2i.py --pipeline-device cuda:0 --trainer-device cuda:1

# Use same device for both (default behavior)
python tests/src/trainer/compare_flux2_i2i.py --pipeline-device cuda:1

# Legacy support (deprecated)
python tests/src/trainer/compare_flux2_i2i.py --device cuda:1
```
