# Version 3.3.7

## Bug Fixes

### Fixed Configuration Validation Errors in All FLUX.2 Config Files
- **Fixed**: Removed unsupported `cache_drop_rate` field from `data.init_args` in all FLUX.2 config files
- **Fixed**: Renamed `validation.validation_steps` to `validation.steps` (correct field name)
- **Fixed**: Renamed `validation.num_validation_samples` to `validation.max_samples` (correct field name)
- **Issue**: Configuration validation was failing with `extra_forbidden` errors for these fields
- **Impact**: All FLUX.2 configuration files can now be loaded successfully without validation errors

## Files Fixed

- `configs/flux2_i2i_fp16_config.yaml`
- `configs/flux2_i2i_fp4_config.yaml`
- `configs/flux2_t2i_fp16_config.yaml`
- `configs/flux2_t2i_fp4_config.yaml`

## Technical Details

The configuration files had three issues:
1. `cache_drop_rate` was not a valid field in `DatasetInitArgs` and has been removed
2. `validation.validation_steps` should be `validation.steps` according to `ValidationConfig` schema
3. `validation.num_validation_samples` should be `validation.max_samples` according to `ValidationConfig` schema

These changes align all FLUX.2 configuration files with the actual Pydantic model definitions in `src/qflux/data/config.py`.
