# Version 3.3.2

## Enhancements

### CUDA Device Selection for Comparison Script
- **Added**: Command-line argument `--device` support in `compare_flux2_i2i.py` test script
- **Feature**: Users can now specify which CUDA device to use (e.g., `--device cuda:1`) to avoid OOM errors
- **Change**: Script now supports:
  - Explicit device via `--device` argument (e.g., `--device cuda:1`)
  - Automatic device selection from `CUDA_VISIBLE_DEVICES` environment variable
  - Device availability validation with memory information display
- **Impact**: Enables running comparison tests on different GPUs when primary GPU is occupied, preventing CUDA OOM errors

## Technical Details

The script now includes a `get_device()` function that:
1. Parses `--device` command-line argument
2. Falls back to `CUDA_VISIBLE_DEVICES` environment variable if no argument provided
3. Validates device availability and displays GPU memory information
4. Provides clear error messages if requested device is not available

Usage examples:
```bash
# Use specific GPU
python tests/src/trainer/compare_flux2_i2i.py --device cuda:1

# Use environment variable
CUDA_VISIBLE_DEVICES=1 python tests/src/trainer/compare_flux2_i2i.py
```
